Introduction¶

Introduction to the Capitularia Digital Edition.

The project runs on three hardware platforms:

Capitularia platforms¶

RRZK Web Projekt¶

An RRZK Web Projekt (https://capitularia.uni-koeln.de) is a standard web hosting package offered by RRZK (Regionales Rechenzentrum der Universität zu Köln).

The RRZK Web Projekt is a package offered by RRZK and consists of an Apache web server and a mysql database. Apache runs a Wordpress installation. We wrote a Wordpress theme and plugins to add the functionality we needed for our project.

RRZK Webprojekt Components¶

Capitularia VM¶

The Capitularia VM (https://api.capitularia.uni-koeln.de) is a root VM also offered by the RRZK. We use the VM for all functionality that is too inconvienent to implement in Wordpress plugins and for all software lacking in the Webprojekt package.

The VM runs a Postgres database server and the Python application server. Next to that it hosts a recent OpenJDK, Saxon and a customized version of CollateX.

Capitularia VM Components¶

Many different XSLT transformations are used to generate the HTML files of the TEI manuscripts and also many auxiliary files like lists of capitularies ans manuscripts.

The Postgres database holds manuscript metadata and the pre-processed text of every chapter in every manuscript.

The app server does collations and offers metadata and fulltext search in the Capitulars.

AFS¶

The AFS Filesystem (/afs/rrz.uni-koeln.de/vol/www/projekt/capitularia/) is a network filesystem.

AFS Components¶

The AFS Filesystem holds all the original manuscript files encoded in TEI and versions thereof converted to HTML. It is accessible from the VM and the Webprojekt. The editors also have access to it through ssh.
