Introduction¶

Introduction to the Capitularia Digital Edition.

The project runs on two hardware platforms:

Capitularia VM¶

The Capitularia VM is a root VM offered by the RRZK.

The VM hosts an Apache Web Server at https://capitularia.uni-koeln.de which runs a Wordpress installation. We wrote a Wordpress theme and many plugins to add the functionality we needed for our project.

The VM runs a Postgres database server and the Python application server. Next to that it hosts a recent OpenJDK, Saxon and a customized version of CollateX.

We use the application server and its API (at https://api.capitularia.uni-koeln.de) for all functionality that is too inconvienent to implement in Wordpress plugins.

Many different XSLT transformations are used to generate the HTML files of the TEI manuscripts and also many auxiliary files like lists of capitularies ans manuscripts.

The Postgres database holds manuscript metadata and the pre-processed text of every chapter in every manuscript.

The app server does collations and offers metadata and fulltext search in the Capitulars.

AFS¶

The AFS Filesystem (/afs/rrz.uni-koeln.de/vol/www/projekt/capitularia/) is a network filesystem.

The AFS Filesystem holds all the original manuscript files encoded in TEI and versions thereof converted to HTML. It is accessible from the VM. The editors also have access to it through ssh.
